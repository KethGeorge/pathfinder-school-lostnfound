# 校园寻路与失物招领系统 — 启动指南

一套「校园地图导航 + 失物招领」的前后端分离项目。后端 Spring Boot，前端纯静态页面（HTML/CSS/JS）。

---

## 一、环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 17+ | 后端基于 Java 17 |
| Maven | 3.6+ | 构建后端 |
| MySQL | 8.0+ | 数据库 |
| Python | 3.x | 仅用于起前端静态服务器（可选） |
| 浏览器 | Chrome/Edge 等现代浏览器 | 打开前端 |

---

## 二、数据库初始化

后端连接的数据库账号见 `campus-backend/src/main/resources/application-dev.yml`：

- 库名：`campus`
- 账号：`campus_user`
- 密码：`campus123`

### 1. 创建数据库账号（首次）

用 MySQL root 登录后执行：

```sql
CREATE USER IF NOT EXISTS 'campus_user'@'localhost' IDENTIFIED BY 'campus123';
GRANT ALL PRIVILEGES ON campus.* TO 'campus_user'@'localhost';
FLUSH PRIVILEGES;
```

### 2. 导入表结构和地图数据

在项目根目录执行（`init.sql` 会自动建库建表）：

```bash
mysql -uroot -p < sql/init.sql
mysql -uroot -p < sql/map_data.sql
```

> `map_data.sql` 含 183 个地图节点（120 路口 + 63 建筑）和 235 条边，是导航寻路的基础数据。

## 三、启动后端

```bash
cd campus-backend
mvn spring-boot:run
```

- 启动端口：**8080**
- 启动成功后访问 http://localhost:8080/api/lostfound/list 应返回 JSON（`code:200`）
- 默认激活 `dev` 环境（`application.yml` 中 `spring.profiles.active: dev`）

打包运行（可选）：

```bash
mvn clean package -DskipTests
java -jar target/campus-system-0.0.1-SNAPSHOT.jar
```

## 四、启动前端

> ⚠️ **前端必须通过 HTTP 服务器打开，不能直接双击 HTML 文件。**
> 直接双击是 `file://` 协议，会导致跨域被拦、地图本地 JSON 读取失败。

前端页面都在**项目根目录**（`index.html` / `map.html` / `个人中心.html` / `失物招领列表.html` / `login.html` / `admin-dashboard.html`）。

### 方式 A：VS Code Live Server 插件（推荐）

安装 Live Server 插件后，右键 `index.html` → **Open with Live Server**。

### 方式 B：Python 静态服务器

在项目根目录执行：

```bash
python -m http.server 5500
```

然后浏览器访问 http://localhost:5500/index.html

## 七、常见问题

**Q: 前端页面能打开但数据加载失败？**
- 确认后端已在 8080 端口运行
- 确认前端是用 HTTP 服务器（`localhost:xxxx`）打开的，不是 `file://`
- 打开浏览器控制台（F12）查看具体报错

**Q: 后端启动报「Port 8080 was already in use」？**
- 8080 端口被占用，先结束占用进程：
  ```bash
  netstat -ano | findstr :8080
  taskkill /F /PID <上一步查到的PID>
  ```

**Q: 后端启动报数据库连接失败？**
- 确认 MySQL 已启动，且已按第二步创建 `campus_user` 账号并导入数据

**Q: 中文数据乱码或「Data too long」？**
- 确认数据库字符集为 `utf8mb4`（`init.sql` 已设置）
